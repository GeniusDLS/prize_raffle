## 🎯 Алгоритм розіграшу

1. **Початок:** Система копіює всіх учасників та розгортає призи за кількістю
2. **Раунд:** 
   - Вибирається учасник (з урахуванням ваги)
   - Вибирається випадковий приз
   - Учасник видаляється зі списку
3. **Повтор:** Раунди продовжуються поки є призи та учасники
4. **Завершення:** Показуються результати

## 🧪 Тестування чесності

### 🔍 Тест послідовностей (Runs Test)
Перевіряє якість генератора випадкових чисел шляхом аналізу послідовностей:

1. **Генерація:** Створюється N випадкових чисел [0,1) за допомогою `crypto.getRandomValues()`
2. **Бінаризація:** Перетворення в 0/1 (менше/більше 0.5)
3. **Підрахунок серій:** Кількість послідовних однакових значень
4. **Статистика:** Z = (R - E(R)) / σ(R)
5. **Висновок:** Порівняння p-value з рівнем значущості (0.05)

**Інтерпретація:**
- p-value > 0.05: Генератор проходить тест (відсутні патерни)
- p-value ≤ 0.05: Підозріло - можливі невипадкові послідовності

### 📊 Тест розподілу (Chi-square Test)
Перевіряє справедливість розподілу призів з урахуванням ваги учасників:

1. **Симуляція:** Проводиться багато віртуальних розіграшів
2. **Підрахунок:** Кількість перемог кожного учасника
3. **Порівняння:** Фактичні результати vs очікувані (за вагою)
4. **Статистика:** χ² = Σ((Oi - Ei)² / Ei)
5. **Висновок:** Порівняння χ² з критичним значенням

**Інтерпретація:**
- χ² ≤ χ²крит: Розподіл справедливий
- χ² > χ²крит: Можлива несправедливість алгоритму

### 🎯 Тест справедливості
Оцінює зважений вибір за кількома компонентами (0-100 балів):

- **Довірчі інтервали (45 балів)**: % учасників у 95% довірчих інтервалах
- **Відхилення Джині (25 балів)**: Різниця між фактичним та очікуваним індексом
- **Середнє відхилення (20 балів)**: Нормалізоване відносне відхилення
- **Консистентність (10 балів)**: Стабільність результатів між учасниками

**Очікувані ймовірності:**
P(учасник) = його_вага / сума_всіх_ваг

**Приклад:**
- Іван (вага 2): 2/6 = 33.3%
- Марія (вага 1): 1/6 = 16.7%
- Петро (вага 3): 3/6 = 50.0%

### Криптографічний генератор
Всі тести використовують `crypto.getRandomValues()` для максимальної якості випадкових чисел:
```javascript
function secureRandom() {
    const array = new Uint32Array(1);
    crypto.getRandomValues(array);
    return array[0] / (0xffffffff + 1);
}
```

## 🎵 Звукові ефекти

### Архітектура звукової системи
Проект використовує гібридний підхід для максимальної сумісності:

1. **Base64 аудіо дані (основний метод)**
   - Звукові файли конвертовані в Base64 та вбудовані в [`js/audio-data.js`](js/audio-data.js)
   - Вирішує CORS проблеми при роботі з `file://` протоколом
   - Забезпечує самодостатність проекту

2. **Web Audio API Fallback**
   - Автоматично активується при помилках завантаження Base64 даних
   - Генерує синтетичні звуки за допомогою осциляторів
   - Гарантує роботу звуків навіть без аудіо файлів

### Типи звукових ефектів

| Звук | Опис | Коли відтворюється |
|------|------|-------------------|
| **🥁 drumSpin** | Звук обертання барабанів | Під час анімації розіграшу |
| **🔔 resultDing** | Короткий сигнал результату | При зупинці барабанів |
| **🎺 victoryFanfare** | Святкові фанфари | При показі вікна переможця |

### Налаштування звуків
- **Увімкнення/вимкнення**: Налаштування → Анімація → "🔊 Звукові ефекти"
- **Автоматична ініціалізація**: При першій взаємодії користувача
- **Гучність**: Автоматично налаштована для кожного типу звуку
- **Зацикленість**: drumSpin зациклюється, інші - одноразові

### Технічна реалізація
```javascript
// Модуль audio-data.js містить Base64 дані
window.AudioData = {
    getAudioData: (soundName) => AUDIO_BASE64_DATA[soundName],
    getAllSounds: () => Object.keys(AUDIO_BASE64_DATA),
    hasSound: (soundName) => soundName in AUDIO_BASE64_DATA
};

// Модуль sound-manager.js керує відтворенням
window.SoundManager = {
    playSpinSound: (duration) => { /* ... */ },
    playResultSound: () => { /* ... */ },
    playVictorySound: () => { /* ... */ }
};
```

### Переваги Base64 підходу
- ✅ **Вирішує CORS проблеми** - працює з локальними файлами
- ✅ **Самодостатність** - не потрібні зовнішні аудіо файли
- ✅ **Швидке завантаження** - дані вбудовані в код
- ✅ **Надійність** - немає залежності від файлової системи
- ✅ **Fallback система** - гарантована робота звуків

## 🐛 Усунення несправностей

### Дані не зберігаються
- Перевірте чи не заблоковано localStorage в браузері
- Очистіть кеш браузера
- Спробуйте інший браузер

### Excel не імпортується
- Переконайтеся що файл має розширення .xlsx або .xls
- Перевірте структуру даних згідно з прикладом
- Спробуйте пересзберегти файл в Excel

### Розіграш не працює
- Додайте принаймні одного учасника та один приз
- Перевірте консоль браузера на помилки
- Перезавантажте сторінку
